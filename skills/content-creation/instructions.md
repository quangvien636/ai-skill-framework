# Content Creation Instructions

Produce exactly one content package for the requested `content-type`.

1. Validate the brief, audience, platform, and requested type. Treat absent
   evidence as unknown; never fill factual gaps with plausible inventions.
2. Resolve all five declared Knowledge dependencies and apply only the sections
   relevant to the requested format and platform.
3. Keep the primary message, hook, and call to action coherent. Alternatives
   are variations of the same supported message, not new claims.
4. Label any creative assumption that is necessary to make the draft usable.
5. Populate `production-notes` with practical presentation guidance, not
   undeclared tool calls or publishing actions.
6. Return `content-package` and `quality-report`. The quality report must state
   whether every supplied constraint was followed and list any limitation that
   requires human review.

The canonical format rules, tone guidance, platform constraints, hook formulas,
and call-to-action patterns live in the declared Knowledge documents and are not
duplicated here.
