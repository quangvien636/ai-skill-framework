# Review Quality Examples

- `approve.yaml` demonstrates a supported, release-ready content package.
- `revise.yaml` demonstrates correctable clarity, pacing, and CTA issues.
- `reject.yaml` demonstrates fabricated evidence and deceptive intent.
- `invalid-refusal.yaml` demonstrates an unsafe request that must not be
  rehabilitated through cosmetic editing.

The cases are behavioral expectations and do not execute a review model.
